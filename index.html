Executive summary

Move AMEC into one governed Microsoft 365 operating model with a controlled migration, collaboration structure, and support path

AMEC is currently operating across multiple countries, domains, mail systems, and storage locations. This proposal consolidates those elements into a single Microsoft 365 model with clearer ownership, cleaner collaboration, baseline security, and a defined path for post-cutover support and governance.


Geographic footprint 3 Operating Countries

Lebanon, Guinea, and Côte d’Ivoire are reflected in the tenant, SharePoint structure, and access model.


Core migration baseline 6 Users / 6 Mailboxes

The implementation baseline covers six licensed employees, six primary mailboxes, and up to fourteen approved email objects.

Local productivity dependency 4 Licenses

Windows and Microsoft Office 2021 remain separate licensing items because Business Basic does not include desktop Office apps.

Commercial path $453.89 / Month

The selected managed transformation path spreads project work, licensing, and selected recurring services across nine months.

What this engagement covers

  • Tenant creation and active-domain onboarding under one Microsoft 365 administration model.
  • Mailbox migration, baseline security configuration, MFA rollout, and local licensing coordination.
  • SharePoint design, initial data cleanup and migration, OneDrive setup, and user training.
  • Post-go-live stabilization followed by optional support, governance, and security service layers.

Engagement storyboard

How the AMEC engagement moves from fragmented operations to a stable Microsoft 365 baseline

The proposal is sequenced deliberately. It starts with validation, then builds the Microsoft foundation, moves into migration and user adoption, and only then expands into recurring support and governance if AMEC approves it.


01 Validate

Lock the baseline before doing the move

Confirm domains, mailboxes, data locations, local devices, source-system readiness, and decision owners so the project is controlled instead of improvised.

02 Build

Create the Microsoft 365 foundation

Set up the tenant, onboard the active domains, provision approved users, assign Business Basic licenses, and prepare desktop licensing for local productivity where needed.

03 Migrate

Move mail and files under governed control

Migrate primary mailboxes, design the SharePoint structure, move approved data from server and Google Drive sources, and align OneDrive usage correctly.

04 Stabilize

Train users, support the cutover, and decide how much management AMEC wants next

Complete user onboarding, resolve early-life issues, and then activate recurring services only where AMEC wants predictable operating support after go-live.

Operational risk Reduced by front-loading validation

The proposal avoids jumping directly into migration before access, data scope, and device realities are confirmed.

Business outcome Cleaner ownership and collaboration

AMEC ends with a more structured email, file, and administration model instead of distributed personal storage and fragmented hosting.

Commercial discipline Core first, optional after

The portal separates mandatory migration work from optional managed services so AMEC can decide how much follow-through it wants.

The portal intentionally mirrors the proposal logic: fix structure first, then add operational maturity layers only where there is clear business value.

Implementation approach

The phased delivery sequence for AMEC

The implementation roadmap is there to stop the project from becoming a loose collection of tasks. Every phase has an activity focus, a dependency, and an expected output.


Phase 1

Discovery and foundation validation

Confirm domains, access, tenant decisions, source data, device realities, and the approved migration baseline before work begins.

Output: validated scope and tenant readiness
Phase 2

Identity, licensing, mailbox migration, and MFA

Create the approved users, migrate the approved mailboxes, align licensing, and switch on baseline protection at the right moment.

Output: Microsoft mail live and security baseline active
Phase 3

SharePoint, data placement, OneDrive, and training

Move the collaboration model into Microsoft so storage behavior and user adoption actually change, not just the mail system.

Output: governed file structure and trained users
Phase 4

Stabilization and optional service onboarding

Resolve early-life issues cleanly, then activate the selected recurring services or keep AMEC on a lighter operating model if preferred.

Output: controlled transition and managed follow-through
